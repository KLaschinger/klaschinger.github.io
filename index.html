CSE 115/116 projects
Finished
Two of my freshman classes for computer science where CSE 115 and CSE 116. In each class, I had to do several coding projects written in Java. For CSE 115, we had to create a game called Keybrick in which the player would learn typing skills by typing a key that would remove an amount of block on the screen based on the amount of connected blocks. In CSE 116, we had to create a computer game based off of the board game "Master Labyrinth". The first part was a team project in which I took on the task of creating the GUI engine. As opposed to what the other teams in the class did by simply using Swing, I took on the task of learning OpenGL to try to expand my coding skills, since at this point I was already proficient in Java. The second part of the project was also a team project, but instead of creating code from scratch, we were given five codebases to pick from. As a team, we had to discuss the merits of each codebase and implement some final features to the game using the codebase we picked.

EzAR-Android
In Starting Phase
For my Sophomore year at UBHacking, I wanted to create an AR app for viewing furniture and how it would look in a room. At around 3am of that Hackathon, me and my partner realized that there was no way that I would be able to finish this app in 24 hours. But I did not want to give up on the idea and thought that there would be many other applications to the idea than the simple furniture idea that my and my partner had. Since there was no library that I could find that would do what I wanted to do, I decided to convert my idea into an Android API that others could use as well.
